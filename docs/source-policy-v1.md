# Source Policy v1

Compute Current prioritizes sources that produce clean evidence about data centers, cloud infrastructure, power and grid constraints, cooling, chips, systems, capital flows, networking, and siting risk.

## Source Handling

- Prefer RSS, Atom, JSON feeds, and public sitemaps.
- Do not scrape aggressively.
- Respect robots, paywall boundaries, and rate limits.
- Classify sources as active feed, active sitemap, landing-page only, blocked, paywalled, extraction failed, or adapter required.

## Single-Source Vendor Policy

Single-source vendor blogs, official blogs, weekly roundups, product updates, and release notes cannot be treated as Core Signal or Cloud Capacity unless the source explicitly supports capacity evidence.

Without that evidence, the allowed public lanes are bounded product, platform, buyer workflow, enterprise architecture, or technical implementation reads.
