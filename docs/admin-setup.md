# Admin Setup

The admin surface is private, noindexed, and disabled until durable production services and
all required authentication variables are configured.

## Required Environment

- `DATABASE_URL`: managed Postgres-compatible connection string.
- `ADMIN_USERNAME`: initial allowlisted username.
- `ADMIN_PASSWORD_HASH`: Argon2id PHC hash generated by the repository helper; never set a
  plaintext password variable. Production rejects legacy or malformed hash formats.
- `ADMIN_SESSION_SECRET`: random 64+ character signing and audit-HMAC secret.
- `ADMIN_ROLE`: `admin` or `editor`. For multiple users, use `ADMIN_USER_ROLES` JSON.
- `ADMIN_MEDIA_PROVIDER=vercel-blob`.
- `BLOB_READ_WRITE_TOKEN`: Vercel Blob read/write token.
- `ADMIN_DATABASE_POOL_SIZE`: optional; defaults to one connection per function instance.

## Provisioning

1. Generate the password hash with `node scripts/admin-password-hash.mjs --password "temporary-password" --dry-run`.
2. Configure the environment values in the preview deployment only.
3. Run `npm run admin:migrate` against the preview database.
4. Deploy the preview and verify login, create/edit/upload/preview/publish/unpublish,
   restart persistence, revision history, deletion, logout, and noindex behavior.
5. Rotate the temporary password and session secret before any production enablement.

Treat password, session-signing, database, and Blob values as one secret rotation boundary.
The Argon2id helper uses OWASP's minimum profile: 19 MiB memory, two iterations, and one lane.

`npm run build` exports the complete published Postgres state into the Astro read model when
`DATABASE_URL` is present. Runtime publish/unpublish actions therefore require a deployment
hook, scheduled rebuild, or CI trigger; the database outbox keeps those transitions replayable.
The read model exists only under ignored `.cache/` runtime storage and must never be committed.
The build only reads the outbox and never acknowledges it. Do not add acknowledgement back to
the build command; a future consumer must verify deployment success and the exact read-model
version first.

Rotating `ADMIN_SESSION_SECRET` invalidates signed cookies. Revoking a session through logout
also persists the revocation in Postgres. Production and Vercel preview runtimes fail closed
when durable auth, database, or Blob configuration is absent.

## Local Development

Outside production/Vercel, the adapter uses `.cache/admin-cms.json` and
`.cache/admin-media/`. These paths are ignored, outside source/build output, and suitable only
for development and tests. Set `ADMIN_SEED_JSON=0` to start with an empty local CMS.
