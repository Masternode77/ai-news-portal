## Production Body Path

The deployed article page renders `expertLensFull.finalArticleBody` first, then falls back to `articleText`, `snippet`, and `summary` in `src/pages/news/[id].astro`.

Current production generation path before this change:

1. `scripts/pipeline.mjs` selects publishable crawled items after extraction QA, relevance routing, taxonomy, and expert-insight gates.
2. `attachExpertLensToVisibleWindow()` calls `attachExpertLens()` from `scripts/lib/expert-lens.mjs`.
3. `generateExpertLensFull()` asks the model for a structured expert-lens JSON payload.
4. `normalizeExpertLensFull()` validates the payload and calls `finalArticleBody()`.
5. If the model output is too short, misses the blueprint, misses expert-insight usage, or contains template language, `finalArticleBody()` falls back to `blueprintFallbackBody()` in `scripts/lib/article-blueprints.mjs`.
6. `blueprintFallbackBody()` previously pulled reusable paragraphs from `buildHumanizedArticleBody()` in `scripts/lib/editorial-humanizer.mjs`.

That means the production body function actually used under failure or fallback conditions was:

- `scripts/lib/expert-lens.mjs` -> `finalArticleBody()`
- fallback: `scripts/lib/article-blueprints.mjs` -> `blueprintFallbackBody()`
- reusable paragraph bank: `scripts/lib/editorial-humanizer.mjs` -> `buildHumanizedArticleBody()`

## Exact Phrase Summary

The repeated strings appeared in two kinds of places:

- Production code and prompt-like fallback surfaces: `scripts/lib/editorial-humanizer.mjs`, `scripts/lib/repetition-detector.mjs`, `AGENTS.md`, `scripts/lib/AGENTS.override.md`, and one repetition test fixture.
- Cached generated records: `src/data/latest-news.json`, `src/data/archived-news.json`, and `src/data/search-index.json`.

The largest production risk was not the prompt alone. It was the fallback path that reused deterministic body paragraphs whenever model output failed validation, plus the cached JSON artifacts that had already stored those paragraphs.

## Remediation Implemented

- Canonical banned phrase inventory moved to `config/bannedPhrases.yml`.
- Production fallback article bodies now route through NarrativeDNA generation instead of reusable paragraph banks.
- Publishing gates now load banned phrases from config and block any generated draft containing a banned phrase.
- Cached public articles are regenerated by `scripts/regenerate-narrative-dna-articles.mjs`.
