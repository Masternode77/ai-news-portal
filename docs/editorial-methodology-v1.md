# Editorial Methodology v1

Compute Current uses an automated editorial workflow to turn source reporting into AI infrastructure intelligence.

## Pipeline

Source item or cluster -> extraction QA -> source scope policy -> evidence pack -> evidence triangulation -> editorial route -> editorial angle -> article thesis -> narrative lede -> section plan -> human analyst draft -> senior editor rewrite -> source fidelity check -> public publish quality gate -> final public article model.

## Quality Standard

- Full articles require clean extraction, clear infrastructure relevance, source-backed facts, a thesis, interpretation, stakeholder impact, a limitation or counterargument, what to watch, and a bottom line.
- Vendor/product posts are scoped to product, platform, buyer workflow, enterprise architecture, or technical implementation implications unless explicit infrastructure capacity evidence is present.
- Items that fail public quality gates are downgraded when possible or quarantined.

## Disclosure

The workflow is AI-assisted. The linked source remains the authoritative record.
