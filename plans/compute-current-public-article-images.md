# Compute Current Public Article Images Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILLS: Use `superpowers:test-driven-development` for any behavior/code change, `superpowers:verification-before-completion` before any completion/commit/push claim, and `editorial-humanizer` if any reader-facing card, dek, alt, or article copy is rewritten. Track steps with checkbox syntax.

**Goal:** Ensure every public Compute Current article/card displays extracted or generated imagery, stock-derived public card imagery is replaced with generated editorial art when needed, and public/internal language plus feed quality gates remain clean before any commit/push.

**Architecture:** Keep the existing brownfield pipeline. Public display flows from `src/data/*.json` through `scripts/lib/homepage-feed-builder.mjs` and `scripts/lib/public-presentation.mjs`, card UI is `src/components/ArticleListCard.astro`, article hero UI is `src/pages/news/[id].astro` plus `src/components/ArticleHeader.astro`, and asset fallback/sync logic is `scripts/lib/article-image-surface.mjs`. Generated assets live under `public/generated/`.

**Tech Stack:** Astro 5, Node ESM scripts, `node:test`, Sharp, existing image providers in `scripts/lib/image-providers/`, npm audit scripts.

---

## Scope And Defaults

- **IN:** Homepage cards, archive cards, search-index public items, taxonomy page items, related article cards, and public longform article routes generated by `src/pages/news/[id].astro`.
- **OUT:** Hidden/quarantined/internal-only records, monetization work, unrelated homepage redesign, weakening content gates, adding dependencies, and publishing internal workflow metadata.
- **Default for "all articles":** Public reader-visible items only. Do not force hidden records into public surfaces.
- **Default for stock replacement:** If stock-derived public card targets exist, replace them with provider-generated editorial art. Do not silently treat local placeholder SVGs as GPT-generated stock replacements.
- **Default if GPT image provider is unavailable:** Stop before commit when stock targets remain. Local SVG fallback is acceptable only for missing non-stock public images or local build continuity.
- **Build caveat:** `npm run build` runs `prepare:static-images`, which can write data/assets. Always record `git status --short` before and after build.
