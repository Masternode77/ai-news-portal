# Source Surface Instructions

## OVERVIEW

`src/` is the Astro public and internal UI surface: pages, components, styles, data JSON, site config, and lightweight display helpers.

## STRUCTURE

```text
src/
|-- components/  # Astro view components for article, archive, admin, and feed surfaces
|-- config/      # public site constants
|-- data/        # pipeline-generated JSON consumed at build time
|-- layouts/     # shared HTML metadata shell
|-- lib/         # display, SEO, and taxonomy helpers
|-- pages/       # public pages, admin pages, RSS/sitemap/robots routes
`-- styles/      # global and public editorial styling
```

## WHERE TO LOOK

| Task | Location | Notes |
|------|----------|-------|
| Homepage/feed UI | `src/pages/index.astro`, `src/components/*Feed*`, `src/components/*Card*` | Preserve dense intelligence-board feel. |
| Article page rendering | `src/pages`, `src/components/Article*`, `src/components/LongformArticleBody.astro` | Keep source attribution and quality disclosures visible. |
| SEO/indexing | `src/layouts/Layout.astro`, `src/lib/seo-safeguards.js`, `src/pages/sitemap.xml.ts` | Admin, dashboard, pricing/sample/briefing, and noindex articles stay out of sitemap. |
| Admin UI pages | `src/pages/admin*`, `src/components/InternalQualityPanel.astro` | Internal pages must be `noindex`. |
| Build-time data | `src/data/*.json` | Usually generated by scripts; avoid hand edits unless doing explicit data repair. |

## CONVENTIONS

- Use Astro components for page UI; keep helper logic in `src/lib` when reused across pages/components.
- Public article/card copy must stay source-specific and infrastructure-relevant.
- Preserve canonical URLs and `noindex` behavior when adding routes.
- Treat `src/data` as pipeline output. If changing schemas, update generators and tests together.
- Styles are global CSS files, not CSS modules or Tailwind.

## ANTI-PATTERNS

- Do not expose admin/debug/internal quality language on public pages.
- Do not hotlink external source images as published card art.
- Do not add public routes that bypass sitemap/noindex policy review.
- Do not turn article pages into generic AI news summaries.
