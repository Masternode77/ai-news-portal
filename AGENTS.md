# PROJECT KNOWLEDGE BASE

## OVERVIEW

Compute Current is an Astro-based AI infrastructure intelligence product. The source tree combines a public static site, a serverless admin editor, and a script-driven editorial pipeline with strict source-fidelity and repetition gates.

## STRUCTURE

```text
.
|-- src/              # Astro pages, components, public data, SEO/display helpers
|-- api/admin/        # Vercel serverless admin/editor endpoints
|-- scripts/          # pipeline, audits, QA runners, content utilities
|-- scripts/lib/      # editorial engine; stricter local override applies
|-- config/           # YAML policy registries and guard inventories
|-- prompts/          # autonomous desk and blog-engine prompt corpus
|-- tests/            # Node test suite for public, editorial, admin, and guard behavior
|-- .github/          # scheduled content refresh and visual QA workflows
|-- public/generated/ # tracked generated image assets
`-- dist/             # build output; do not edit by hand
```

## WHERE TO LOOK

| Task | Location | Notes |
|------|----------|-------|
| Public page/layout change | `src/pages`, `src/components`, `src/styles` | Follow `src/AGENTS.md`; preserve noindex/admin separation. |
| Article/card rendering | `src/components`, `src/lib/publicPresentation.js` | Source data comes from `src/data/*.json`. |
| Admin editor/API work | `api/admin`, `src/pages/admin*` | GitHub API persistence and cookie auth; follow `api/AGENTS.md`. |
| Content generation or QA logic | `scripts/lib` | Also follow `scripts/lib/AGENTS.override.md`. |
| Pipeline orchestration | `scripts/pipeline.mjs`, `scripts/run-*.mjs` | Builds and workflows call package scripts directly. |
| Editorial policy or phrase gates | `config`, `scripts/lib/*guard*`, `tests/*guard*` | Update config and publishing guard together. |
| Prompt behavior | `prompts`, `scripts/lib/*writer*`, `scripts/lib/*router*` | Prompt changes must respect product-fit and source-fidelity gates. |
| Verification surface | `tests`, `scripts/audit-*.mjs`, `scripts/qa-*.mjs` | Use targeted tests first, `content:gate` for release confidence. |
| Generated article/image data | `src/data`, `public/generated` | Generated by scripts/CI; do not hand-edit unless the task is data repair. |

## PRODUCT DEFINITION

Compute Current is not a generic AI blog. It is AI infrastructure decision-support for operators, investors, cloud capacity teams, data center developers, and infrastructure strategists.

Every generated or edited article must answer:

1. What changed?
2. Why does it matter for AI infrastructure?
3. Who benefits?
4. Who is exposed?
5. What bottleneck or decision point should readers watch?

Do not force weakly related AI news into data center infrastructure framing.

## EDITORIAL HARD RULES

- Never generate a long-form article from a source that fails extraction QA.
- Never reuse the same article structure for consecutive items.
- Every article must have a source-specific thesis.
- Do not make infrastructure claims unsupported by extracted source text, feed snippet, or explicit source metadata.
- Do not add fallback language reusable unchanged across many articles.
- Always run repetition and source-fidelity evals before publishing or marking article copy publishable.
- Do not weaken extraction QA, repetition checks, source-fidelity checks, or product-fit boundaries without explicit user request.

## BANNED GENERIC PATTERNS

The canonical inventory lives in `config/bannedPhrases.yml`. Update that config and the publishing guard together instead of copying blocked strings into prompts, fallback copy, fixtures, or article data.

## COMMANDS

```bash
npm run dev
npm run check
npm run build
npm run test
npm run test:quality-gates
npm run test:blog-engine-v4
npm run test:autonomous-desk
npm run content:gate
npm run qa:qc
```

`npm run build` runs `prepare:static-images` and then `astro build`. Runtime dashboard data is never generated into `public/`. `content:gate` is the broad release gate: check, build, selected tests, public audits, image audit, and admin exclusion audit.

## GENERATED AND RUNTIME PATHS

- `dist/`, `.astro/`, `evidence/`, `artifacts/`, `.omo/`, `.omx/`, `.vercel/`, `.cache/`, `outputs/`, `.worktrees/`, and logs are build/runtime/evidence surfaces.
- `public/generated/` is generated image output but may be tracked by the pipeline workflow.
- Do not treat generated output as source of truth when source modules or policy files exist.

## NOTES

- `scripts/lib/AGENTS.md` and `scripts/lib/AGENTS.override.md` are authoritative for content-generation modules.
- No ESLint, Prettier, Jest, Vitest, or Playwright config files are present; tests use Node's built-in runner and visual QA scripts import Playwright directly.
- GitHub Actions uses a push-back content refresh model; Vercel builds from `npm run build`.
- Admin writes are GitHub API commits, not database mutations.
